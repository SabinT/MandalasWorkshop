Radial Repetition

How to tile a motif around a ring using ring()

🔵
Overview

The ring() function is the core tool for creating mandala patterns. It takes a motif — a small drawing described in a simple "easy space" coordinate system — and repeats it n times evenly around a circular ring.

Every repetition is automatically rotated and mapped from the flat easy space grid (x ∈ [−S, S], y ∈ [−S, S]) into its correct position in polar coordinates on the canvas. The size S defaults to 1 and can be changed with setDesignSpace(S).

🔄
ring( opts )

Repeats a motif function evenly around a circular ring on the canvas.

ring({ shape, n, r1, r2 })
Parameter Type Description
shape function A motif function that calls motif drawing commands (mLine, mBezier, mCircle, etc.). It will be called once to capture the drawing commands, which are then replayed for each repetition around the ring.
n number Number of repetitions. The full 360° circle is divided evenly into n segments and the motif is drawn once in each. Typical values: 6, 8, 12, 16, 24.
r1 number (pixels) Inner radius — distance from the canvas centre to the inner edge of the ring, in pixels. Corresponds to y = −S in motif space (where S is the design-space size).
r2 number (pixels) Outer radius — distance from the canvas centre to the outer edge of the ring, in pixels. Corresponds to y = +S in motif space (where S is the design-space size). Must be greater than r1.

Example

// Draw myMotif 12 times around a ring
// between radius 80 px (inner) and 160 px (outer).
ring({ shape: myMotif, n: 12, r1: 80, r2: 160 });
💡 The canvas origin (0, 0) is the centre of the ring. In p5.js you can move the drawing origin with translate(width/2, height/2) inside draw() so rings are automatically centred.
📐
Easy Space (Motif Coordinate System)

Motifs are designed in a square called easy space: both x and y range from −S to +S, where S is the design-space size (default 1). Call setDesignSpace(S) in setup() to use a different scale — for example setDesignSpace(100) lets you work in a −100 to 100 range.

When ring() maps a motif into a ring segment the axes have the following meaning:

Axis Range Maps to …
x −S … +S Angular position within the ring segment. −S is the left edge of the segment, 0 is its centre, +S is the right edge.
y −S … +S Radial position within the ring. −S is the inner radius (r1), +S is the outer radius (r2).
💡 Use showMotif(myMotifFn) to preview your motif in an interactive debugger that shows the easy-space grid with axis labels, zoom, and pan.
📏
setDesignSpace( size )

Sets the half-extent of the easy-space coordinate system used by all motif drawing functions. Call this once in setup() before drawing.

setDesignSpace(size)
Parameter Type Description
size number Half-extent of the design space. Coordinates will run from −size to +size on both axes. Defaults to 1 (so coordinates range from −1 to +1).

Example — working in a −100 to 100 coordinate space

const DESIGN_SPACE = 100;

function setup() {
  createCanvas(1000, 1000);
  setDesignSpace(DESIGN_SPACE);  // motif coords now range -100 to 100
}

function motifLeaf() {
  mLine(0, -80, 0, 80);  // coordinates scaled to design space
}
💡 showMotif() automatically adapts its grid labels to the current design-space size, so the axis labels always show the correct coordinate values.
⭕
Building Concentric Rings

Call ring() multiple times with different radii and repetition counts to build up layered mandala patterns.

Example — three concentric rings

function draw() {
  background(20);
  translate(width / 2, height / 2);

  stroke(255);
  noFill();

  // Inner ring — 8 petals, thin band
  ring({ shape: motifPetal, n: 8,  r1: 40,  r2: 100 });

  // Middle ring — 12 leaves, wider band
  ring({ shape: motifLeaf,  n: 12, r1: 100, r2: 190 });

  // Outer ring — 24 dots, thin outer band
  ring({ shape: motifDot,   n: 24, r1: 190, r2: 220 });
}
